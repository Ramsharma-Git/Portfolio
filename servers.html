Deploying Servers

How to Deploy an Apache Server

Apache is a popular open-source web server that can be deployed on a variety of operating systems, including Linux, Windows, and macOS. In this blog post, we will walk you through the steps on how to deploy an Apache server on a Linux machine.


1. Install Apache : The first step is to install Apache on your Linux machine. To do this, open a terminal window and run the following command on debian based Linux distributions like Ubuntu, Debian, etc:

[ubuntu@server ~]# sudo apt install apache2 -y

If you are using rpm based linux distribution such as Centos, RHEL, Amazon Linux,etc. Then run the following command: 

[root@server ~]# sudo yum install httpd -y

2. Enable Apache : To start the Apache Server, use the following command: 

[ubuntu@server ~]# sudo systemctl enable --now apache2

On RPM based Distributions, run the following command:

[root@server ~]# sudo systemctl enable --now httpd

3. Test Apache : Now the Apache Service is running, you can test it to make sure that it is working properly. To do this, run curl localhost command or open a web browser and navigate to http://localhost. If you see the Apache Welcome page, then it is working properly.

If you do not see the welcome page, make sure that the apache service is running and port 80 is allowed.


4. Deploying Simple Website : To deploy your own website, you can put your code into the Document Root of Apache which is /var/www/html. Make sure that your index.html file is present in the Document Root.

How to deploy a private DNS Server

A private DNS server is a DNS server that is used to resolve domain names within a private network. This can be useful for a number of reasons, such as improving performance and security. By deploying your private DNS server, you can have more control over your network's DNS queries and responses.


1. Install BIND : The first step is to update your system and install a DNS software on your system. To do this, run the following command on Ubuntu/Debian :

[ubuntu@server ~]# sudo apt update -y
sudo apt install bind9 -y

On RPM based Distributions, run the following command:

[root@server ~]# sudo yum update -y
sudo yum install bind -y

2. Configure BIND : After installing BIND, we need to configure it using the /etc/named.conf file. Add the following lines in the configuration file : 

options{
directory "/var/named";
recursion no;
};
zone "ramsharma.site" IN{
type master;
file "ram";
};

The "options" field containes the document root of the DNS which is /var/named. The "zone" field contains the Domain Name and Zone file in which all the DNS records are listed. In my case, I have my Domain name "ramsharma.site" and the zone file name is "ram".


3. Create DNS Zone file : We have to create a zone file in the /var/named directory with the same name which is mentioned in the /etc/named.conf file. In my case, I am creating a file named ram and adding the DNS records in it : 

$TTL 1M 
$TTL 1M
@ IN SOA @ rname.invalid. (
                                0 ; serial
                                1D ; refresh
                                1H ; retry
                                1W ; expire
                                3H ) ; minimum
@          IN        NS               ns1.ramsharma.site.
@          IN        NS               ns2.ramsharma.site.
ns1        IN        A                13.232.100.7
ns2        IN        A                13.232.100.7
@          IN        A                17.20.120.40

4. Start the DNS Service : After making correct entry in configuration file and creating a zone file, we have to start the DNS service in your system by the following command :

[root@server ~]# sudo systemctl enable --now named

The DNS service uses the Port 53. Make sure that the port is allowed. If all the entries are correct then the DNS service should start. If your named service is not starting then there is a possibility that the entry made in configuration file are not correct.

How to Get Free SSL Certificate

Running your Website with a Secure Connection requires obtaining a SSL Certificate. The SSL Certificate is provided by the Certificate Authority (CA). Generally we have to purchase these certificates in order to use them, But we can also get it for free with Let's Encrypt. Let's Encrypt is a trusted CA that provides free SSL certificates for securing websites. Here are the steps to get a free SSL certificate from Let's Encrypt :


1. Insert Certbot (Let's Encrypt Client) : Certbot is an official tool provided by Let's Encrypt to obtain and manage SSL certificates automatically. I am asuming that you have already configured your webserver with your domain name. Also allow the Port 443 on your system. Now to install Certbot on your server, run the following command : 

[ubuntu@server ~]# sudo apt install certbot python3-certbot-apache #For Apache
[ubuntu@server ~]# sudo apt install certbot python3-certbot-nginx #For NGINX

2. Obtain SSL Certificate : In order to obtain the certificate, simplly run the following command : 

[ubuntu@server ~]# sudo certbot --apache #For Apache
[ubuntu@server ~]# sudo certbot --nginx #For NGINX

Certbot will then prompt you to enter an email address for renewal and security notifications, and you will be asked if you agree to the terms of service. After agreeing, Certbot will automatically configure SSL and obtain the certificate for your domain.

3. Automating Certificate Renewal : Let's Encrypt certificates are valid for some days, so it's essential to renew them periodically. Certbot automatically sets up a cron job for certificate renewal. However, it's a good practice to test the renewal process before relying on the automated renewal. You can test the renewal with the dry run command : 

[ubuntu@server ~]# sudo certbot renew --dry-run

If everything works correctly, you should see a message indicating that the dry run was successful.

Deploying a Database Server

A database server is a specialized computer system or software application that provides database management services to other computers or applications on a network. To deploy a database server, you'll need to install the MariaDB/MySQL server package. Here is the step-by-step guide to configure database server :

1. Install the MySQL/MariaDB package : Run the following command to install the MariaDB server package :

[ubuntu@server ~]# sudo apt update -y
sudo apt install mariadb-server -y

2. Start and Enable MariaDB : After installation, start the MariaDB service and enable it to start on boot by the folowing command :

[ubuntu@server ~]# sudo systemctl enable --now mariadb

3. Secure MariaDB Installation : We must do the secure installlation of MariaDB to set a root password, remove anonymous users, allow or disallow root login remotely, and more. We can do this by running the secure installation script as follows :

[ubuntu@server ~]# sudo mysql_secure_installation

4. Accessing MySQL : To access the MySQL shell as the root user, use the following command and enter the root password when prompted :

[ubuntu@server ~]# sudo mysql -u root -p
